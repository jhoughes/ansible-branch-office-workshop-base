# Output Captures

This directory holds **real terminal captures** from running the workshop's playbooks against a real lab environment. These captures serve two purposes:

1. **Reference for attendees.** Each `LAB-X.Y.md` guide points at a capture file in this directory and says "your output should look like this." If an attendee's output is meaningfully different, that's a clue something went wrong.
2. **Integration testing for instructors.** Doing the capture pass before each workshop *is* the lab integration test. If a section breaks while capturing, you fix it before workshop day.

> **Important:** These captures cannot be generated by AI assistants. They have to be captured during a real run by a human against a real lab. See `HOW-TO-CAPTURE.md` for the protocol.

## Why placeholders instead of waiting?

The lab guides reference these files by name. Having placeholders in the repo from day one means:

- The lab guides have working links from the start
- The directory structure is obvious to anyone exploring the repo
- It's clear which captures are pending vs. complete (placeholders end in `.txt.placeholder`)
- Workshop instructors immediately know what they need to capture

After the build pass, the workflow is: replace each `.txt.placeholder` with the real `.txt`, commit, done.
